# Python 가상 환경 생성

python3.10을 새로 설치한 후, 프로젝트 폴더 내에서 가상 환경을 생성하는 과정에서 다음과 같은 에러가 발생했다.

```bash
Error: Command ‘[‘/home/mars1/flask/env/bin/python3’, ‘-Im’, ‘ensurepip’, ‘–upgrade’, ‘–default-pip’]’ returned non-zero exit status 1.
```

알아보니 위 에러가 발생하는 이유는 `pip`와 같은 명령을 실행할 수 있는 파일이 없어서 그런 것이라고 한다. 따라서 **`pip` 없이 가상 환경을 생성한 후에 따로 `pip`를 설치하는 방식으로 진행해야 한다.**

## PIP 없이 가상 환경 생성

`pip` 없이 가상 환경을 생성하려면, `--without-pip` 옵션을 추가해 주면 된다.

```bash
python -m venv 가상환경이름 --without-pip
```

## 생성된 가상 환경에 PIP 설치

가상 환경이 문제없이 생성되었다면 이제 `pip`를 설치해주자.

import { Steps } from "nextra-theme-docs";

<Steps>

### 가상 환경 활성화

```bash
source ./가상환경이름/bin/activate
```

### PIP 설치

```bash
curl https://bootstrap.pypa.io/get-pip.py | python
```

</Steps>

## PIP 업그레이드 및 다운그레이드

`pip`는 꽤 빈번하게 업데이트 되기 때문에 수시로 업그레이드를 해주어야 한다. `pip`명령어를 사용할 때 알림 로그로 알려주기도 하니까 너무 신경쓸 필요는 없다.

```bash
pip install --upgrade pip
```

프로젝트에 의해서 `pip`버전을 다운그레이드 해야하는 경우도 생기는데, 이 경우는 `pip`로 `pip`의 낮은 버전을 설치하면 된다.

```bash
pip install pip=='pip 특정 버전'
```

끝.
